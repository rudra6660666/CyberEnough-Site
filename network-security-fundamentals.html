Module 1: Network Architecture and Protocols

Build a solid foundation in network fundamentals and security protocols.

OSI Model and TCP/IP Stack

Networks are the backbone of modern communication, and understanding how they work is your first step toward mastering network security. In this section, we’ll dive deep into two foundational frameworks: the OSI Model and the TCP/IP Stack. These models are like architectural blueprints for how data moves across networks, from your smartphone to a server halfway across the world. By the end, you’ll understand how networks operate, where vulnerabilities lurk, and how to protect them.

The OSI Model: A Seven-Layer Framework

The OSI (Open Systems Interconnection) Model, developed by the International Organization for Standardization, breaks down network communication into seven distinct layers. Each layer handles a specific part of the process, making it easier to understand and troubleshoot networks. Here’s a detailed look at each layer:

  • Layer 1: Physical Layer - This is the hardware layer, dealing with cables, switches, and electrical signals. It’s where bits (0s and 1s) are transmitted over physical media like Ethernet cables or Wi-Fi signals. Security risks here include physical tampering (e.g., someone plugging a malicious device into your network). You’ll learn how to secure physical access points, like locking server rooms.
  • Layer 2: Data Link Layer - This layer handles communication between devices on the same network, using MAC addresses. It’s responsible for error detection and framing data into packets. Vulnerabilities include MAC spoofing, where attackers pretend to be a trusted device. We’ll explore defenses like port security on switches.
  • Layer 3: Network Layer - This layer manages routing, using IP addresses to send data across different networks. It’s where routers operate, deciding the best path for data. Security threats include IP spoofing and routing attacks. You’ll learn about secure routing protocols and IP address filtering.
  • Layer 4: Transport Layer - Responsible for reliable data delivery, this layer uses protocols like TCP and UDP. TCP ensures data arrives correctly, while UDP prioritizes speed. Attacks like SYN flooding target this layer. You’ll study how to configure TCP settings for security and monitor for anomalies.
  • Layer 5: Session Layer - This layer manages sessions between applications, ensuring data streams are maintained. Weak session management can lead to session hijacking. You’ll learn about session timeouts and secure authentication.
  • Layer 6: Presentation Layer - This layer translates data into a format the application layer can use, handling encryption and compression. Insecure encryption (e.g., outdated SSL) is a common vulnerability. You’ll explore modern encryption standards like TLS.
  • Layer 7: Application Layer - The top layer, where user applications (like browsers) interact with the network. Threats include malware and phishing. You’ll learn about securing applications with HTTPS and input validation.

The TCP/IP Stack: The Internet’s Blueprint

The TCP/IP Stack is a practical, four-layer model that powers the internet. It simplifies the OSI Model but covers similar concepts. Here’s how it works:

  • Link Layer - Combines the OSI’s Physical and Data Link layers, handling hardware communication and local network traffic. You’ll learn about Ethernet and Wi-Fi security, like WPA3 for wireless networks.
  • Internet Layer - Equivalent to the OSI’s Network layer, this uses IP (IPv4 or IPv6) to route data across networks. You’ll study IP addressing, subnetting, and defenses against attacks like IP fragmentation.
  • Transport Layer - Like the OSI’s Transport layer, it uses TCP and UDP for data delivery. You’ll explore how TCP’s three-way handshake works and how to mitigate denial-of-service (DoS) attacks.
  • Application Layer - Combines the OSI’s Session, Presentation, and Application layers, handling user-facing protocols like HTTP and DNS. You’ll learn to secure these with encryption and authentication.

Security Across the Layers

Each layer has unique vulnerabilities, but also specific defenses. For example, at the Physical layer, you might use tamper-proof hardware, while at the Application layer, you’d implement secure coding practices. You’ll explore real-world attacks, like ARP spoofing (Data Link) or SQL injection (Application), and how to counter them with tools like packet analyzers (e.g., Wireshark) and intrusion detection systems.

Hands-On Learning

This section includes practical exercises to solidify your understanding. You’ll analyze network traffic using Wireshark to see how data moves through each layer, simulate a simple IP spoofing attack in a lab environment, and configure basic security measures like MAC filtering. By mapping real network processes to the OSI and TCP/IP models, you’ll gain a clear picture of how networks function and where to apply security.

Why This Matters

Mastering the OSI Model and TCP/IP Stack gives you a mental map of networking, essential for identifying and mitigating security risks. Whether you’re troubleshooting a network issue or defending against a cyberattack, these models are your guide. By the end of this section, you’ll be ready to tackle network security challenges with confidence!

What You’ll Learn:

  • The seven layers of the OSI Model and their roles in networking.
  • The four layers of the TCP/IP Stack and how they power the internet.
  • Common security vulnerabilities at each layer and how to defend against them.
  • Practical skills like analyzing network traffic and configuring basic security settings.

Network Protocols and Security

Network protocols are the rules that let devices communicate, like a universal language for the internet. In this section, you’ll explore key protocols, how they work, the risks they face, and how to secure them. This is critical knowledge for any cybersecurity pro, as protocols are both the backbone of networks and prime targets for attackers.

What Are Network Protocols?

Protocols define how data is formatted, transmitted, and received. Think of them as the etiquette of network communication. Here are the key protocols you’ll study:

  • HTTP/HTTPS - HyperText Transfer Protocol (HTTP) powers web browsing, while HTTPS adds encryption via TLS. HTTP is vulnerable to eavesdropping, so HTTPS is critical for secure browsing.
  • DNS - Domain Name System translates domain names (like google.com) into IP addresses. DNS spoofing can redirect users to malicious sites.
  • FTP/SFTP - File Transfer Protocol (FTP) transfers files, but it’s unencrypted. Secure FTP (SFTP) uses SSH for secure transfers.
  • SMTP/POP3/IMAP - These email protocols (Simple Mail Transfer Protocol, Post Office Protocol, and Internet Message Access Protocol) handle sending and receiving emails. Unencrypted email can expose sensitive data.
  • ICMP - Internet Control Message Protocol is used for diagnostics (e.g., ping). It’s often exploited in ping flood attacks.

Vulnerabilities in Protocols

Each protocol has weaknesses that hackers exploit. For example:

  • HTTP - Without HTTPS, data is sent in plain text, making it easy to intercept (e.g., via man-in-the-middle attacks).
  • DNS - DNS cache poisoning can redirect users to fake websites. DNSSEC (Domain Name System Security Extensions) helps prevent this.
  • FTP - Unencrypted credentials can be sniffed. SFTP or FTPS (FTP over TLS) mitigates this.
  • SMTP - Email spoofing allows attackers to send fake emails. SPF, DKIM, and DMARC are defenses to verify senders.
  • ICMP - Ping floods can overwhelm networks, a type of DoS attack. Limiting ICMP traffic can help.

Securing Protocols

You’ll learn practical ways to secure these protocols:

  • Encryption - Use HTTPS, SFTP, or TLS-enabled email to encrypt data in transit.
  • Authentication - Implement strong credentials and multi-factor authentication (MFA) for protocols like FTP.
  • Configuration - Harden protocol settings, like disabling unused services or restricting ICMP traffic.
  • Monitoring - Use tools like Wireshark to detect suspicious protocol activity, such as unexpected DNS queries.

Hands-On Learning

This section includes labs to bring protocols to life. You’ll set up a secure HTTPS website using a free TLS certificate, simulate a DNS spoofing attack in a controlled environment, and configure an SFTP server. You’ll also analyze protocol traffic with Wireshark to identify vulnerabilities, like unencrypted FTP sessions, and apply fixes.

Why This Matters

Protocols are the lifeblood of networks, but they’re also entry points for attacks. Understanding their mechanics and securing them is a core cybersecurity skill. By the end of this section, you’ll be able to identify risky protocols, apply security measures, and troubleshoot protocol-related issues.

What You’ll Learn:

  • Key network protocols (HTTP, DNS, FTP, etc.) and their roles.
  • Common protocol vulnerabilities and attack techniques.
  • Practical security measures, like encryption and authentication.
  • Hands-on skills in configuring and monitoring protocols.

Module 2: Firewalls and Intrusion Detection Systems

Learn about network security devices and how they protect against threats.

Firewall Technologies

Firewalls are your network’s gatekeepers, controlling who gets in and out. In this section, you’ll master firewall technologies, from basic setups to advanced systems, and learn how they protect networks from threats. Think of firewalls as the bouncers of your network, checking every packet’s ID before letting it through.

What Are Firewalls?

A firewall is a hardware or software device that filters network traffic based on predefined rules. It sits between your network and the outside world, blocking malicious traffic like malware, hackers, or unauthorized access attempts. Firewalls operate at different OSI layers, depending on their type:

  • Packet-Filtering Firewalls (Layer 3) - These inspect packet headers (e.g., IP addresses, ports) and allow or block based on rules. They’re fast but basic, missing deeper threats like application-layer attacks.
  • Stateful Inspection Firewalls (Layer 4) - These track the state of connections (e.g., established TCP sessions), offering more context than packet-filtering. They’re common in small networks.
  • Application-Layer Firewalls (Layer 7) - Also called proxy firewalls, these inspect application data (e.g., HTTP requests). They’re slower but great for blocking sophisticated attacks.
  • Next-Generation Firewalls (NGFW) - These combine packet filtering, stateful inspection, and application-layer analysis with features like intrusion prevention and deep packet inspection (DPI).

How Firewalls Protect Networks

Firewalls enforce security policies by allowing or denying traffic. For example, you might allow port 80 (HTTP) for web traffic but block port 23 (Telnet) to prevent insecure access. They protect against:

  • Unauthorized Access - Blocking external devices from accessing internal servers.
  • Malware - Stopping malicious packets from spreading viruses.
  • DoS Attacks - Limiting excessive traffic to prevent network overload.

Configuring Firewalls

You’ll learn to create firewall rules, such as:

  • Allowing specific IP ranges (e.g., your company’s network).
  • Blocking risky ports (e.g., port 445, often used by ransomware).
  • Setting up DMZ (demilitarized zone) rules for public-facing servers.

We’ll also cover firewall management, including logging, monitoring, and updating rules to adapt to new threats.

Hands-On Learning

In labs, you’ll configure a software firewall (e.g., pfSense or Windows Defender Firewall) to protect a small network. You’ll create rules to block unauthorized access, simulate a DoS attack to test your rules, and analyze firewall logs to identify threats. You’ll also explore NGFW features like DPI using open-source tools.

Why This Matters

Firewalls are a cornerstone of network security, and knowing how to deploy and manage them is essential. This section equips you with practical skills to design and maintain firewall defenses, preparing you for real-world cybersecurity roles.

What You’ll Learn:

  • The types of firewalls and their strengths.
  • How to create and manage firewall rules.
  • Practical applications in securing networks.
  • Hands-on experience with firewall configuration and monitoring.

IDS/IPS Implementation

Intrusion Detection and Prevention Systems (IDS/IPS) are your network’s security cameras, constantly watching for suspicious activity and stopping threats in real-time. In this section, you’ll learn how IDS and IPS work, how to implement them, and how they complement firewalls to keep networks safe.

What Are IDS and IPS?

An IDS (Intrusion Detection System) monitors network traffic for signs of malicious activity, like unusual patterns or known attack signatures. It alerts administrators but doesn’t block traffic. An IPS (Intrusion Prevention System) goes further, actively blocking threats. Both operate at various OSI layers, analyzing packets or application data.

  • Network-Based IDS/IPS (NIDS/NIPS) - Monitors entire network segments, detecting attacks like port scans or malware.
  • Host-Based IDS/IPS (HIDS/HIPS) - Runs on individual devices, watching for suspicious activity like file changes.

How IDS/IPS Work

IDS/IPS use two main detection methods:

  • Signature-Based Detection - Matches traffic against a database of known attack patterns (e.g., a specific malware signature).
  • Anomaly-Based Detection - Identifies deviations from normal traffic patterns, catching new or unknown threats.

IPS can block threats by dropping packets, resetting connections, or redirecting traffic. Popular tools include Snort, Suricata, and commercial solutions like Cisco Secure IPS.

Implementing IDS/IPS

Setting up an IDS/IPS involves:

  • Sensor Placement - Positioning sensors at key network points, like between your internal network and the internet.
  • Rule Configuration - Writing rules to detect specific threats, like SQL injection attempts or brute-force attacks.
  • Alert Management - Analyzing alerts to distinguish real threats from false positives.
  • Tuning - Adjusting rules to reduce false positives and optimize performance.

Hands-On Learning

In labs, you’ll install an open-source IDS/IPS like Snort, configure rules to detect common attacks (e.g., a port scan), and analyze alerts. You’ll simulate attacks in a safe environment to see how IPS blocks them and practice tuning rules to balance security and performance.

Why This Matters

IDS/IPS catch threats that firewalls miss, making them critical for layered security. This section prepares you to deploy and manage these systems, giving you skills to protect networks from advanced attacks.

What You’ll Learn:

  • The differences between IDS and IPS and their roles.
  • How to install and configure IDS/IPS tools.
  • Techniques for analyzing and responding to alerts.
  • Hands-on experience with IDS/IPS deployment and tuning.
